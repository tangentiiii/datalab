# Data Lab 实验报告

## 1. `bitXor`

### 题目要求

只使用 `~` 和 `&` 实现 `x ^ y`。

### 实现代码（带注释）

```c
int bitXor(int x, int y) {
  /* 利用德摩根律把异或改写为只含 ~ 和 & 的形式
   * x ^ y = (x & ~y) | (~x & y)
   *       = ~ ( ~(x & ~y) & ~(~x & y) )
   */
  return ~((~(x & ~y)) & (~(~x & y)));
}
```

### 解题思路

异或在某一位上为 1，当且仅当这一位上 `x` 和 `y` 不同。  
因此可以先写出标准形式：

- `x & ~y` 表示“`x` 为 1、`y` 为 0”的位；
- `~x & y` 表示“`x` 为 0、`y` 为 1”的位；
- 两者按位或就是异或。

由于题目不允许使用 `|`，再利用德摩根律：

- `a | b = ~(~a & ~b)`

即可得到最终结果。

### 简要证明

对任意一位分别讨论：

- 若两位相同，则 `(x & ~y)` 与 `(~x & y)` 都为 0，结果为 0；
- 若两位不同，则二者恰有一个为 1，结果为 1。

所以该表达式与 `x ^ y` 等价。

### 总操作数

`8`

## 2. `tmin`

### 题目要求

返回 32 位补码整数的最小值。

### 实现代码（带注释）

```c
int tmin(void) {
  /* 补码最小值的符号位为 1，其余位为 0 */
  return 1 << 31;
}
```

### 解题思路

32 位补码最小值是 `1000...000`，即只有最高位为 1。  
因此将 `1` 左移 31 位即可。

### 简要证明

`1 << 31` 的二进制形式正是 `0x80000000`，这就是 32 位补码的最小值。

### 总操作数

`1`

## 3. `isTmax`

### 题目要求

判断 `x` 是否为 32 位补码最大值 `Tmax`。

### 实现代码（带注释）

```c
int isTmax(int x) {
  /* Tmax 满足：
   * y = x + 1 = Tmin
   * x ^ y = 0xFFFFFFFF
   * 但 x = -1 时也满足 x ^ (x + 1) = 0xFFFFFFFF，
   * 所以还要排除 y == 0 的情况
   */
  int y = x + 1;
  return !(~(x ^ y)) & !!y;
}
```

### 解题思路

对于 `Tmax = 0x7fffffff`，加一后会变成 `Tmin = 0x80000000`。  
这两个数按位异或的结果是全 1，因此可用：

- `x ^ (x + 1) == 0xFFFFFFFF`

来识别。  
但 `x = -1` 时，`-1 + 1 = 0`，同样也有：

- `-1 ^ 0 = 0xFFFFFFFF`

所以必须额外要求 `x + 1 != 0`。

### 简要证明

若 `x` 为 `Tmax`，则条件显然成立。  
若 `x ^ (x + 1)` 为全 1，则说明 `x + 1` 与 `x` 每一位都相反。满足这一性质的补码整数只有：

- `0111...111` 与 `1000...000`，即 `Tmax`；
- `1111...111` 与 `0000...000`，即 `-1`。

再用 `!!y` 排除 `y = 0`，故仅在 `x = Tmax` 时返回 1。

### 总操作数

`7`

## 4. `allOddBits`

### 题目要求

判断所有奇数位是否都为 1。

### 实现代码（带注释）

```c
int allOddBits(int x) {
  /* 构造 0xAAAAAAAA，奇数位全为 1 */
  int mask = 0xAA;
  mask = mask | (mask << 8);
  mask = mask | (mask << 16);

  /* 只保留 x 的奇数位，再与 mask 比较 */
  return !(mask ^ (x & mask));
}
```

### 解题思路

需要检查的是第 1、3、5、...、31 位是否全为 1。  
先构造掩码 `0xAAAAAAAA`，其所有奇数位为 1、偶数位为 0。然后：

- `x & mask` 可以提取出 `x` 的所有奇数位；
- 若这些位全为 1，则应与 `mask` 完全相同。

所以判断 `mask ^ (x & mask)` 是否为 0 即可。

### 简要证明

若 `x` 的每个奇数位都为 1，则 `(x & mask) = mask`，异或为 0，返回 1。  
只要有任意一个奇数位不是 1，那么 `(x & mask)` 在该位上为 0，与 `mask` 不同，异或非 0，返回 0。

### 总操作数

`7`

int negate(int x) {

- `x + (~x + 1) = 0`

故 `~x + 1` 正是 `-x`。

## 6. `isAsciiDigit`

### 题目要求

判断 `x` 是否在 ASCII 字符 `'0'` 到 `'9'` 的范围内，即 `0x30 <= x <= 0x39`。

### 实现代码（带注释）

```c
int isAsciiDigit(int x) {
  /* 0x30~0x37 的高 29 位与 0x30 完全一致
   * 0x38~0x39 与 0x38 在除最低 1 位外完全一致
   */
  int a = (x ^ 0x30) >> 3;
  int b = (x ^ 0x38) >> 1;

  /* 命中 0x30~0x37 或 0x38~0x39 即可 */
  return (!a) | (!b);
}
```

### 解题思路

ASCII 数字分成两段看：

- `0x30 ~ 0x37`：与 `0x30` 只有低 3 位可能不同，所以 `(x ^ 0x30) >> 3 == 0`；
- `0x38 ~ 0x39`：与 `0x38` 只有最低 1 位可能不同，所以 `(x ^ 0x38) >> 1 == 0`。

只要满足其中一段即可。

### 简要证明

若 `x` 在 `0x30~0x37`，则它与 `0x30` 的差别仅限最低 3 位，右移 3 位后必为 0。  
若 `x` 在 `0x38~0x39`，则它与 `0x38` 的差别仅限最低 1 位，右移 1 位后必为 0。  
反之，若这两个条件都不满足，则 `x` 不可能落在 `0x30~0x39` 内。

### 总操作数

`7`

## 7. `conditional`

### 题目要求

实现与 `x ? y : z` 相同的效果。

### 实现代码（带注释）

```c
int conditional(int x, int y, int z) {
  /* 当 x != 0 时：
   * !x = 0
   * y_mask = 0xFFFFFFFF
   * z_mask = 0x00000000
   *
   * 当 x == 0 时：
   * !x = 1
   * y_mask = 0x00000000
   * z_mask = 0xFFFFFFFF
   */
  int y_mask = ~(!x) + ((!x) << 1);
  int z_mask = ~(!x) + 1;

  return (y & y_mask) | (z & z_mask);
}
```

### 解题思路

核心是把条件选择写成“掩码混合”的形式：

- 当 `x != 0` 时选 `y`；
- 当 `x == 0` 时选 `z`。

只要构造两个互补掩码即可：

- `y_mask` 在需要选择 `y` 时为全 1，否则为全 0；
- `z_mask` 与之相反。

最后返回：

- `(y & y_mask) | (z & z_mask)`

### 简要证明

当 `x != 0` 时，`y_mask = 0xFFFFFFFF`，`z_mask = 0`，结果为 `y`。  
当 `x == 0` 时，`y_mask = 0`，`z_mask = 0xFFFFFFFF`，结果为 `z`。  
因此与三目运算完全等价。

### 总操作数

`11`

## 8. `isLessOrEqual`

### 题目要求

若 `x <= y` 返回 1，否则返回 0。

### 实现代码（带注释）

```c
int isLessOrEqual(int x, int y) {
  /* 先处理相等情况 */
  int cond = !(x ^ y);

  /* 提取符号位 */
  int sign_x = (x >> 31) & 1;
  int sign_y = (y >> 31) & 1;
  int diff_sign = sign_x ^ sign_y;

  /* 这里的 diff 实际是 x - y */
  int diff = x + (~y) + 1;

  /* 若符号不同，则 x 为负、y 为正时一定有 x <= y */
  cond = cond | (diff_sign & sign_x);

  /* 若符号相同，则 x - y <= 0 等价于最高位为 1 或等于 0
   * 由于相等情况前面已经处理过，这里只需看负号
   */
  cond = cond | (!diff_sign & !!((diff >> 31) & 1));

  return cond;
}
```

### 解题思路

这道题难点在于避免减法溢出。分情况讨论最安全：

1. 若 `x == y`，直接返回 1。
2. 若 `x` 和 `y` 符号不同：
   - `x` 为负、`y` 为正，一定有 `x <= y`；
   - `x` 为正、`y` 为负，一定有 `x > y`。
3. 若 `x` 和 `y` 符号相同：
   - 此时 `x - y` 不会因为异号相减而溢出；
   - 只要看 `x - y` 的符号即可。

### 简要证明

符号不同的两个补码数，负数一定小于正数，这一结论无需做减法。  
符号相同时，`x - y` 的符号真实反映了大小关系：

- `x - y < 0`，则 `x < y`；
- `x - y = 0`，则 `x = y`；
- `x - y > 0`，则 `x > y`。

代码正是按这三类情况分别判断，因此正确。

### 总操作数

`18`

## 9. `logicalNeg`

### 题目要求

不用 `!` 实现逻辑非。

### 实现代码（带注释）

```c
int logicalNeg(int x) {
  /* 只有 x == 0 时，x 和 -x 的符号位都为 0
   * 其余非零数中，x 和 -x 至少有一个符号位为 1
   */
  int sign_x = (x >> 31) & 1;
  int sign_neg_x = ((~x + 1) >> 31) & 1;
  int temp = sign_neg_x | sign_x;

  /* temp 为 0 时返回 1，否则返回 0 */
  return ~temp + 2;
}
```

### 解题思路

对任意非零补码整数：

- 若 `x > 0`，则 `x` 的符号位为 0，而 `-x` 的符号位为 1；
- 若 `x < 0`，则 `x` 的符号位为 1。

因此只要 `x != 0`，`x` 与 `-x` 至少有一个符号位是 1。  
而 `x = 0` 时，`-x` 仍为 0，两者符号位都为 0。  
据此即可区分零和非零。

### 简要证明

`temp = sign(x) | sign(-x)`：

- 当 `x = 0`，`temp = 0`；
- 当 `x != 0`，`temp = 1`。

最后用 `~temp + 2` 将 `0/1` 翻转成 `1/0`，所以结果等于逻辑非。

### 总操作数

`9`

## 10. `howManyBits`

### 题目要求

返回用补码表示整数 `x` 所需的最少位数。

### 实现代码（带注释）

```c
int howManyBits(int x) {
  int ans = 1;      /* 先计入符号位 */
  int mask = x >> 31;
  int next_x = 0;
  int z_mask = 0;
  int y_mask = 0;

  /* 若 x 为负，则转成 ~x；
   * 这样正数和负数都统一成“找最高有效 1 的位置”
   */
  x = (x & ~mask) | (~x & mask);

  /* 下面是二分查找最高位 1 所在区间 */
  next_x = x >> 16;
  z_mask = ~(!next_x) + 1;
  y_mask = ~z_mask;
  x = (next_x & y_mask) | (x & z_mask);
  ans = ans + (16 & y_mask);

  next_x = x >> 8;
  z_mask = ~(!next_x) + 1;
  y_mask = ~z_mask;
  x = (next_x & y_mask) | (x & z_mask);
  ans = ans + (8 & y_mask);

  next_x = x >> 4;
  z_mask = ~(!next_x) + 1;
  y_mask = ~z_mask;
  x = (next_x & y_mask) | (x & z_mask);
  ans = ans + (4 & y_mask);

  next_x = x >> 2;
  z_mask = ~(!next_x) + 1;
  y_mask = ~z_mask;
  x = (next_x & y_mask) | (x & z_mask);
  ans = ans + (2 & y_mask);

  next_x = x >> 1;
  z_mask = ~(!next_x) + 1;
  y_mask = ~z_mask;
  x = (next_x & y_mask) | (x & z_mask);
  ans = ans + (1 & y_mask);

  /* 最后若当前 x 非 0，再补上最后 1 位 */
  z_mask = ~(!x) + 1;
  y_mask = ~z_mask;
  ans = ans + (1 & y_mask);

  return ans;
}
```

### 解题思路

补码最少位数 = 符号位 + 数值部分最高有效位的位置。

关键观察：

- 对正数，直接找最高的 1；
- 对负数，找 `~x` 中最高的 1 即可。

为什么对负数要取反？  
因为一个负数用最少位表示时，本质上取决于它与全 1 前缀首次不同的位置。例如：

- `-1` 写作 `1`，只需要 1 位；
- `-5` 写作 `1011`，只需要 4 位。

把负数取反后，就能把问题统一成“找最高 1 的位置”。

之后代码采用类似二分查找的方法：

1. 先看高 16 位是否非 0；
2. 再看接下来的 8 位；
3. 再看 4 位、2 位、1 位。

每一步如果高半部分非 0，就把答案加上对应偏移量，并把 `x` 更新到高半部分继续查找。

### 简要证明

设变换后的 `x` 为：

- 原数非负时仍为 `x`；
- 原数为负时变为 `~x`。

那么最终需要的位数就是：

- `1 + floor(log2(x)) + 1`（当 `x != 0`）

其中前一个 `1` 是符号位，后一个 `1` 是最高有效位本身。  
二分过程逐步定位最高有效 1 的位置，且每一步都只在“高半部分是否存在 1”这一事实上做分支编码，因此最终得到的正是最高位位置，从而得到最少位数。

对特殊值也成立：

- `x = 0` 时只需 1 位；
- `x = -1` 时 `~x = 0`，也只需 1 位；
- `x = Tmin` 时最高位必须保留，共需 32 位。

### 总操作数

`62`

## 11. `floatScale2`

### 题目要求

返回浮点数 `f` 的位级表示 `2*f`。

### 实现代码（带注释）

```c
unsigned floatScale2(unsigned uf) {
  /* 若阶码全 1，则是 NaN 或无穷，直接返回 */
  if ((uf & 0x7f800000) == 0x7f800000)
    return uf;

  /* 若是非规格化数，符号位不变，尾数左移 1 位 */
  if (((uf >> 23) << 24) == 0) {
    return (uf & 0xff800000) | ((uf & 0x007fffff) << 1);
  }

  /* 规格化数乘 2 等价于阶码加 1 */
  return uf + 0x00800000;
}
```

### 解题思路

IEEE 754 单精度浮点数分三类处理：

1. `exp = 255`：表示 NaN 或无穷，按题意原样返回；
2. `exp = 0`：表示非规格化数，乘 2 只需把小数部分左移 1 位；
3. 其余情况：表示规格化数，乘 2 只需把阶码加 1。

### 简要证明

浮点数表示为：

- 规格化：`(-1)^s * 1.f * 2^E`
- 非规格化：`(-1)^s * 0.f * 2^(1-Bias)`

对规格化数乘 2，本质上是指数 `E` 加 1；  
对非规格化数乘 2，本质上是小数部分整体左移。  
代码恰好与这两种定义对应，因此正确。

### 总操作数

`10`

## 12. `floatFloat2Int`

### 题目要求

返回 `(int) f` 的位级结果；若超范围、NaN 或无穷，返回 `0x80000000u`。

### 实现代码（带注释）

```c
int floatFloat2Int(unsigned uf) {
  int sign = uf & 0x80000000;
  int exp = (uf & 0x7f800000) >> 23;
  int E = exp - 127;
  int frac = uf & 0x007fffff;
  int M = frac;

  /* 规格化数需要补回隐藏的最高位 1 */
  if (exp)
    M = frac + 0x00800000;

  if (E >= 0) {
    if (E <= 23)
      M = M >> (23 - E);
    else if (E <= 30)
      M = M << (E - 23);
    else
      return 0x80000000u;
  } else {
    /* 绝对值小于 1，强制转换后为 0 */
    return 0;
  }

  /* 恢复符号 */
  if (sign)
    M = ~M + 1;

  return M;
}
```

### 解题思路

先把浮点数拆成：

- 符号位 `sign`
- 阶码 `exp`
- 尾数 `frac`

对于规格化数，其有效数字是 `1.frac`，因此需要补上隐藏位 `1`。  
再根据真实指数 `E = exp - 127` 分类：

1. `E < 0`：绝对值小于 1，转成整数后为 0；
2. `0 <= E <= 23`：有效数右移得到整数部分；
3. `24 <= E <= 30`：左移扩展成更大的整数；
4. `E > 30`：超出 32 位有符号整数范围，返回溢出标记。

最后根据符号位决定是否取负。

### 简要证明

浮点数转整数时，本质上就是取：

- `(-1)^s * M * 2^(E-23)`

其中 `M` 为补上隐藏位后的 24 位有效数。  
`E` 的不同范围只会影响该式应当左移还是右移，以及是否越界。  
代码对这几类情况逐一覆盖，因此与 C 的强制类型转换规则一致。

### 总操作数

`15`（按代码中的算术、位运算与比较手工统计）

## 13. `floatPower2`

### 题目要求

返回浮点数 `2.0^x` 的位级表示。

### 实现代码（带注释）

```c
unsigned floatPower2(int x) {
  /* 过大，返回 +INF */
  if (x >= 0x00000080)
    return 0x7f800000u;

  /* 能表示成规格化数 */
  else if (x >= -126)
    return (127 + x) << 23;

  /* 能表示成非规格化数 */
  else if (x >= -149)
    return 0x00000001 << (x + 149);

  /* 过小，下溢为 0 */
  return 0;
}
```

### 解题思路

单精度浮点数中：

- 规格化数的指数范围是 `E in [-126, 127]`；
- 非规格化数能继续表示到 `2^-149`；
- 再小就只能下溢为 0。

因此分三段讨论：

1. `x > 127`：结果太大，返回正无穷；
2. `-126 <= x <= 127`：可表示为规格化数，阶码字段为 `x + 127`；
3. `-149 <= x < -126`：可表示为非规格化数，只需在小数部分放置一个对应的 1；
4. `x < -149`：结果过小，返回 0。

### 简要证明

`2^x` 的有效数字恒为 `1.0`。  
因此：

- 若是规格化数，只需设置阶码 `exp = x + Bias`，尾数全 0；
- 若是非规格化数，则其数值为 `frac * 2^-149`，故令 `frac = 1 << (x + 149)` 即可；
- 超范围时分别映射到 `+INF` 与 `0`。

这与 IEEE 754 单精度编码规则完全一致。

### 总操作数

`7`
